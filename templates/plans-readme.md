# Plans Directory

This directory contains implementation plans for the project.

## Conventions

- Each plan should have a descriptive name: `YYYY-MM-DD-feature-name.md`
- Include clear acceptance criteria
- Define testing strategy (unit tests or BDD)
- Mark the status: DRAFT, IN_PROGRESS, COMPLETED, ARCHIVED

## Creating a New Plan

1. Copy `_template.md` to a new file with the naming convention above
2. Fill in all required sections
3. Get approval before starting implementation
4. Update status as you progress

## Plan Statuses

| Status | Description |
|--------|-------------|
| DRAFT | Plan is being written, not yet approved |
| IN_PROGRESS | Implementation has started |
| COMPLETED | Implementation finished and verified |
| ARCHIVED | Plan is no longer relevant or was superseded |

## Template

See `_template.md` for the recommended format.
